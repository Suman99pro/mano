# mano

> `man` + AI examples — practical, real-world usage examples for any terminal
> command, generated by a local LLaMA-family model and grounded in the real
> man page.

Ever run `man tar` and get lost in flags with no idea which combination
you actually need? `mano` asks a local LLM to generate concrete, ready-to-run
examples for the command — using the real man page as grounding so it
doesn't invent flags that don't exist.

```
$ mano tar
Generating examples for 'tar' using model 'llama3.2'...
  $ tar -czvf archive.tar.gz ./my-folder
      Compress a folder into a gzipped tarball
  $ tar -xzvf archive.tar.gz -C /tmp
      Extract a gzipped tarball into /tmp
  $ tar -tvf archive.tar
      List contents of a tarball without extracting
  ...
```

## Features

- 🧠 Grounded in the real `man` page (NAME/SYNOPSIS/DESCRIPTION) so examples
  stay close to the command's actual flags
- 💾 Caches results locally so repeat lookups are instant
- 🔒 Fully local — runs via [Ollama](https://ollama.com), no API keys, no
  data leaves your machine
- 🔧 Works with any Ollama model (`llama3.2`, `llama3.1:8b`, `mistral`, etc.)

## Requirements

- Linux or macOS with `man` and `col` (both standard on virtually every
  distro)
- [Ollama](https://ollama.com) installed and running
- At least one model pulled, e.g. `ollama pull llama3.2`

## Installation

```bash
git clone https://github.com/Suman99pro/mano.git
cd mano
chmod +x mano
sudo mv mano /usr/local/bin/mano
```

Or, one-liner if you just want the script itself:

```bash
curl -o /usr/local/bin/mano https://raw.githubusercontent.com/<your-username>/mano/main/mano
sudo chmod +x /usr/local/bin/mano
```

## Usage

```bash
mano <command>            # show cached or freshly generated examples
mano -r <command>         # force regenerate, ignore cache
mano -m <model> <command> # use a specific Ollama model (default: llama3.2)
```

Environment variables:

| Variable         | Description                          | Default            |
|------------------|---------------------------------------|---------------------|
| `MANO_MODEL`     | Default Ollama model to use            | `llama3.2`          |
| `MANO_CACHE_DIR` | Where generated examples are cached    | `~/.cache/mano`     |

## How it works

1. Runs `man <command>` and extracts the first ~120 lines (NAME, SYNOPSIS,
   DESCRIPTION, and usually the start of OPTIONS)
2. Sends that excerpt plus the command name to your local model, asking for
   5-8 realistic usage examples in a strict, greppable format
3. Caches the result to `~/.cache/mano/<command>__<model>.txt` so you don't
   pay the inference cost twice for the same command/model pair

## Limitations

- Quality depends on the model you pull. Smaller models (3B) are fast but can
  be shallow; `llama3.1:8b` or `mistral` tend to give better examples if your
  machine can handle the extra compute.
- If a command has no man page, `mano` will still try, but falls back to the
  model's general knowledge with a warning.
- Not a replacement for reading the real man page for full accuracy — treat
  the output as a helpful starting point, not gospel.
